Endpoints

Available Resources

All endpoints return JSON and are accessible under the /api prefix. Interactive documentation is available via Swagger UI.

🍽  Menu

GET
GET /api/menu All items
GET /api/menu?category=… By category

Browse the full menu or filter by category: Starters, Mains, Dal & Curry, Breads, Desserts, Drinks, Specials.

Try it →

👨‍🍳  Chefs

GET
GET /api/chefs All chefs

Returns all active chef profiles with name, title, bio, specialty and image URL, ordered by seniority.

Try it →

📅  Reservations

POST
POST /api/reservations Create booking

Submit a table reservation with guest name, email, date, time and party size. Returns confirmation ID.

View schema →

🛒  Orders

POST
POST /api/orders Place order

Place a food order with customer details and line items. Validates menu item availability and calculates totals.

View schema →

⭐  Reviews

GET
GET /api/reviews Approved reviews

Retrieves all guest-approved reviews with star rating, comment, customer name, and source platform.

Try it →

✉️  Contact

POST
POST /api/contact Send message

Submit a contact enquiry. Validated for name, email and message content. Stored for staff follow-up.

View schema →

Base URL

Getting Started

All API requests are made to the base URL below. Replace :port with 8080 in local development.

http://localhost:8080/api

Example request:

curl http://localhost:8080/api/menu
